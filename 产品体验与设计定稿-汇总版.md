# AI 海报项目 · 产品体验与设计定稿（汇总版）

> **本文档定位**：把当前讨论中所有设计理念收敛为**一份可执行定稿**。  
> 与 `用户视角产品体验改造方案.md` 的关系：本文件为**总纲**；后者可作历史/拆分说明，执行以本文件为准。

---

## 一、一页原则（North Star）

1. **价值一句话**：输入一句话，先得到**可编辑**的第一版海报，再精修，再导出。  
2. **路径最短**：从「打开产品」到「开始生成」≤ **1 次点击**；首屏只说清一件事。  
3. **先完成再进阶**：默认只暴露「主题 + 行业 + 尺寸」，其余进「高级设置」。  
4. **失败可恢复**：报错用人话；**不丢用户已填内容**；给明确重试/下一步。  
5. **专业感底线**：无乱码、术语统一、加载与空态可读、少「后台感」。

---

## 二、价值主张（对用户怎么说）

- **主标题（对外统一）**：`输入一句话，30 秒生成可编辑海报`  
- **副解释（三句以内）**：  
  - 比「纯 AI 出图」多一步**可控**：字、色、主图、背景都能改。  
  - 比「空白画布」省时间：先出初稿，再排版细调。  
  - 能落地：支持 **PNG / JPG / PDF** 与常见社媒尺寸。

---

## 三、竞品打法怎么「合成」到本产品

| 来源 | 学什么 | 在本产品怎么用 |
|------|--------|----------------|
| **Canva 类** | 首屏大标题 + 一句价值 + **一个主 CTA**；下面再铺能力 | **欢迎页 / 官网首屏**：左文案右产品图，主按钮「立即生成海报」 |
| **稿定 / 创客贴 类** | **场景优先**、AI 写成一排「玩法」入口 | **欢迎页第二屏**：6 张以内场景卡，点进 `/ai-poster` 带 `preset` |
| **通用 SaaS** | 信任条、重复 CTA、区块标题清晰 | 首屏下：三步说明 + 价值三卡 + 页尾再 CTA |

**不做**：初创阶段不要学稿定「顶栏 mega-menu 铺满」；主路径保持 **欢迎页 → AI → 编辑器** 三条即可。

## 四、欢迎页信息架构（定稿线框）

### 4.1 顶栏

- 左：Logo + 产品名  
- 中（可选）：`AI 生成` · `模板` · `帮助`  
- 右：主按钮 `立即生成` → `/ai-poster`；次入口 `从模板开始` → `/home`（可带默认 `tempid`）

### 4.2 Hero（首屏 · Canva 式）

- 眉标（可选）：`智能海报 · 可编辑导出`  
- **H1**：`输入一句话，30 秒生成可编辑海报`  
- **副标题**：`先出第一版，再微调文字、配色和主图，最后一键导出`  
- **大号主题输入框** + 主按钮 `立即生成海报` + 次按钮/链接 `从模板开始`  
- **右列**：真实产品界面截图（表单 + 预览），比插画更贴「工具」

**路由**：主 CTA 跳转 `/ai-poster`（可用 query 预填 `theme`）；次 CTA 进模板页。

### 4.3 第二屏（稿定式 · 玩法 / 场景）

- 标题：`选一个场景，快速开始`  
- **≤6 张卡片**：活动主视觉、招聘、电商促销、课程、餐饮上新、节日（与 `AiPoster.vue` preset 对齐）  
- 每张：场景名 + 一行说明 + `用这个开始` → `/ai-poster?preset=xxx`

### 4.4 三步 + 价值三卡 + 信任条 + 页尾 CTA

- **三步**：输入主题 → 生成预览 → 导出或进编辑器精修  
- **三卡**：快（十几秒）· 可控（非死图）· 能导出（PNG/JPG/PDF）  
- **信任条（可选）**：商用/版权、本地部署、常见尺寸等**只写属实的一句**  
- **页尾**：`现在生成你的第一张海报` + 按钮 + `查看模板`

## 五、`/ai-poster` 页面（定稿交互）

### 5.1 新手默认露出

- **仅三项**：主题、行业、尺寸  
- **用途 / 风格 / 二维码 / 长文案** → 「展开高级设置」  
- 预设场景：首屏 **3 个热门**，其余「更多场景」（降选择焦虑）

### 5.2 主操作与布局

- **全局仅一个实心主色主按钮**在首屏逻辑中心；底部 **sticky 主按钮**（滚动仍可见）  
- 避免双「生成预览」权重并列：顶栏保留「导出」+ 弱化的「生成」，侧栏底部为**主生成**  
- **微调顺序（定稿）**：更新文案 → 更新配色 → 更新主图 → 更新背景 → 尺寸适配  

### 5.3 预览区与空态

- 深色预览区 **提亮约 8–12%**，空态正文对比度足够（浅灰主字 + 更深灰辅助）  
- 空态标题：`输入主题后，约 10–30 秒生成第 1 版`  
- 附 **3 条可点击示例 prompt**，一键填入主题  
- 生成后顶部或预览下方 **固定下一步条**：`应用到编辑器精修` · `导出`

### 5.4 视觉舒适（相对「后台感」的修正）

- 左栏卡片：**边框更弱**（浅描边 + 轻阴影二选一为主）  
- **字号阶梯统一**：区块标题 > 字段标签 > 辅助说明，减少「满屏小字」  
- 可选：**最近一次预览缩略条**，减轻「怕改坏」的焦虑

## 六、视觉风格规范（定稿 Token）

### 6.1 气质

清爽、可信、**创作工具**感；浅色大面 + **少量**氛围渐变；**一种主强调色**贯穿全站。

### 6.2 色彩（与现有 AI 页对齐时推荐）

- **主色 Primary**：`#ea580c`～`#f97316`（橙）  
- **主色 Hover**：比主色略深 5–8%  
- **页面背景**：浅灰或渐变，例 `linear-gradient(180deg, #f7f3ec 0%, #edf4fb 100%)`  
- **卡片**：`rgba(255,255,255,0.82)` + `1px` 半透明边，**弱玻璃**（blur 克制）  
- **主文案**：`#0f172a` / `#1e293b`  
- **次要文案**：`#64748b`  
- **边框**：`rgba(148, 163, 184, 0.24)`  

### 6.3 字体与字号（桌面）

- 中文：`"PingFang SC", "Microsoft YaHei", system-ui, sans-serif`  
- Hero 标题：`36–44px` / `font-weight: 700` / `line-height: 1.15`  
- 区块标题：`22–28px` / `600`  
- 正文：`15–16px` / `line-height: 1.6`  
- 辅助：`13–14px` / `#64748b`  
- 移动端 Hero 降至约 `28–32px`

### 6.4 圆角、阴影、间距

- 大卡片圆角：**20px**（与 AI 页 panel 统一）  
- 按钮：**12px** 或 **全圆角**，全站二选一  
- 卡片阴影：`0 12px 40px rgba(15, 23, 42, 0.06)` 一类，**忌多层重阴影**  
- 区块纵向间距：**72–96px**；内容区 `max-width: 1120–1200px` 居中  

### 6.5 动效

仅保留按钮 hover **轻微上移 + 阴影略强**（150–200ms）；避免花哨动效。

### 6.6 Element Plus 落地提示

- 通过 CSS 变量统一 `--el-color-primary` 与主色  
- Hero 输入框 **加高**（44–48px）、圆角 12px，与主 CTA 同级  
- 次要操作用 `plain` / `text`，不与主按钮抢视觉

## 七、文案与术语（定稿表）

### 7.1 按钮

| 场景 | 文案 |
|------|------|
| 主生成 | 立即生成海报 |
| 再次生成 | 重新生成 |
| 进编辑器 | 应用到编辑器精修 |
| 导出 | 导出 PNG / JPG / PDF |
| 模板入口 | 从模板开始 |
| 顶栏 | 立即生成（可选） |

### 7.2 状态与失败

- 生成中：`正在生成第 1 版海报（约 10–30 秒）`（可分步文案：文案 → 配色 → 版式，若实现）  
- 成功：`第 1 版已生成，可继续微调`  
- 生成失败：`生成失败，请重试（已保留你的输入）`  
- 保存失败：`保存失败，请稍后重试`（技术细节进日志或「查看详情」）  
- 离开确认：`当前修改尚未应用，离开后将丢失，确定离开吗？`  

### 7.3 术语与乱码（P0）

- 统一：`模板`（不用「模版」）  
- 统一：`加载中...`（不用「拼命加载中」）  
- **修复乱码**：尤其图层/组件名中的 **`二维码`** 等（查模板 JSON、i18n、接口返回）

## 九、优先级与排期（定稿）

### P0（必须）

1. 欢迎页 / 首屏路径（≤1 点击到 AI）  
2. AI 页新手模式 + 空态与预览区可读性  
3. 乱码与术语统一  
4. 错误提示用户化 + 失败不丢输入  

### P1（建议）

微调按钮顺序、预设精简、顶栏一级「AI 生成」、AI 页 sticky 主按钮、左栏弱边框与字号阶梯  

### P2（可选）

`/create` 别名、空态示例词、最近一次缩略预览、埋点与 A/B（事件名另表）  

**两天冲刺**：Day1 P0 路径+AI 新手+文案；Day2 P1 体验+回归+校对。

## 十、验收标准（上线前）

- 首屏价值与主 CTA 清晰；**无乱码、术语统一**  
- 新用户 **3 分钟内**可完成「生成 → 导出或进编辑器」  
- 失败场景可重试且**输入保留**  
- 欢迎页与 AI 页 **主色、圆角、字号阶梯**一致，整体不像拼凑后台页  

---

## 十一、上线后指标（1 周）

首页 → 首次生成转化率、首次会话生成成功率、首次导出率、失败后重试率、首次完成时长（下降为目标）。
