鲲穹海报 / poster-design 宝塔部署说明

一、这次部署重点修复了什么
1. 默认国内镜像已统一到更稳的配置：
   - Docker Hub：docker.m.daocloud.io
   - npm：https://registry.npmmirror.com/
   - pip：https://mirrors.aliyun.com/pypi/simple/
   - Debian apt：mirrors.aliyun.com
   - rembg 模型下载代理：REMBG_MODEL_MIRROR=https://ghproxy.net/
2. api 镜像已补齐 Python3、rembg、onnxruntime、中文字体 fonts-noto-cjk、fontconfig。
3. api 镜像已补齐 service/scripts/cutout_worker.py，避免线上抠图时报脚本不存在。
4. api 容器已新增 U2NET_HOME=/app/.u2net，并挂载独立卷缓存模型，避免每次重建后重新拉模型。

二、本地正常、线上异常最常见的差异
1. 本地开发通常是 Vite 直接代理接口，线上则多一层宝塔反代和容器内 Nginx。
2. 本地系统常自带中文字体，线上容器若没装 CJK 字体，海报截图、封面、Puppeteer 渲染容易乱码或字体回退。
3. 本地直接访问 localhost 时接口可通，线上若把域名误反代到 api:7001，会导致 SPA 路由、/oauth/callback、/static、/api/screenshots 链路异常。
4. 本地 rembg 可能已缓存模型，线上首次部署若不走国内镜像，模型下载会卡在 GitHub。

三、宝塔正确接法
1. Docker 只对外暴露 web 容器映射端口，也就是 deploy/.env 里的 HTTP_PUBLISH_PORT。
2. 宝塔网站反向代理目标必须填：
   http://127.0.0.1:HTTP_PUBLISH_PORT
   例如：
   http://127.0.0.1:8098
3. 不要把整站反代到 7001。
4. 不要只反代 /api。这个项目依赖整站反代，容器内 Nginx 会继续转发：
   - /api
   - /design
   - /auth
   - /admin
   - /ai
   - /usage
   - /static
   - /oauth/callback

四、部署步骤
1. 进入目录：
   cd /www/wwwroot/poster/poster-design/deploy
2. 用 env.template 生成 .env 并填写数据库、域名、OAuth、AI 密钥。
3. 首次或镜像源改动后，建议强制重建：
   docker compose -f docker-compose.yaml down
   docker builder prune -af
   docker compose -f docker-compose.yaml build --no-cache api web
   docker compose -f docker-compose.yaml up -d
4. 宝塔站点开启反向代理，目标填写：
   http://127.0.0.1:HTTP_PUBLISH_PORT
5. HTTPS 场景下，宝塔反代需保留 Host 和 X-Forwarded-Proto。

五、上线前自检
1. 检查 compose 配置是否展开正确：
   docker compose -f docker-compose.yaml config
2. 检查首页是否经由 web 容器返回：
   curl -I http://127.0.0.1:HTTP_PUBLISH_PORT/
3. 检查 OAuth 回调页是否返回 index.html，而不是 404：
   curl -I http://127.0.0.1:HTTP_PUBLISH_PORT/oauth/callback
4. 检查接口是否通过容器内 Nginx 转发：
   curl -I http://127.0.0.1:HTTP_PUBLISH_PORT/design/list
5. 检查中文字体和 Chromium：
   docker compose exec api fc-list | grep "Noto Sans CJK"
   docker compose exec api chromium --version
6. 检查 rembg 与模型缓存目录：
   docker compose exec api python3 -c "from rembg import new_session; print(bool(new_session('u2net')))"
   docker compose exec api ls -la /app/.u2net

六、如果你之前遇到过这些问题，对应怎么判断
1. `apt-get update` 报 `bookworm Release 404`：
   说明镜像源路径配错了，尤其是把 Tencent 路径写成了 `/debian/debian` 这种形式。现在默认已改成阿里云 Bookworm 可用路径。
2. 首页或回调页打不开：
   基本先查宝塔是否把域名反代到了 7001，或者只代理了 /api。
3. 线上海报文字乱码、本地正常：
   先查容器内中文字体是否存在，再查实际使用的字体文件是不是前端外链且被浏览器回退。
4. 抠图第一次很慢或卡死：
   先查 REMBG_MODEL_MIRROR 是否生效，再查 /app/.u2net 卷里是否已有模型缓存。

七、建议的宝塔上线策略
1. 域名整站反代到 web 端口，不做静态和接口拆分。
2. 宝塔只负责 HTTPS、证书和外层反代，应用路由交给容器内 Nginx。
3. 更新部署包后固定执行一次 `docker compose build --no-cache api web`，避免沿用旧的错误 apt layer。
