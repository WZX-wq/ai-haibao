# 无服务器构建部署说明

目标：在有 Docker 的构建机上提前构建好镜像，服务器只负责加载镜像并启动，不再执行 `npm ci`、`vite build`、`webpack build`。

## 生成预构建部署包

在构建机项目根目录执行：

```powershell
powershell -ExecutionPolicy Bypass -File deploy\pack-prebuilt-image-bundle.ps1
```

生成文件：

- `deploy/poster-design-prebuilt-images-deploy.zip`

## 服务器部署

1. 上传 `poster-design-prebuilt-images-deploy.zip`
2. 解压：

```bash
unzip -oq poster-design-prebuilt-images-deploy.zip
cd prebuilt-image-bundle
```

3. 配置环境变量：

```bash
cp env.template .env
```

4. 加载镜像：

```bash
docker load -i images/poster-design-api-prebuilt.tar
docker load -i images/poster-design-web-prebuilt.tar
```

5. 启动服务：

```bash
docker compose up -d
```

## 特点

- 服务器不参与源码构建
- 不再执行 `npm ci`
- 首次上线速度主要取决于上传 zip 和 `docker load`
- 更适合当前后端依赖安装较慢的场景
