```bash

```

### 启动服务
```bash
docker compose up -d
```

### 测试抠图功能
1. 访问编辑器
2. 上传一张图片
3. 点击"抠图"按钮
4. **首次使用会自动下载模型（约 10-30 秒）**
5. 后续抠图速度：5-15 秒/张

## 📊 技术细节

### 使用的模型
- **模型名称**: u2net（高质量版本）
- **模型大小**: 约 176MB
- **下载方式**: 首次使用时自动下载
- **存储位置**: 容器内 `/root/.u2net/` 目录

### 性能指标
- **内存需求**: 至少 2GB，推荐 3GB
- **处理速度**: 5-15 秒/张（CPU 版本）
- **首次抠图**: 30-60 秒（包含模型下载时间，176MB）

### 抠图质量
- **边缘质量**: 高（u2net 是高质量模型）
- **适用场景**: 生产环境、高质量要求
- **降级选项**: 可切换到 u2netp 获得更快速度（但质量略低）

## 🔧 可选配置

如果需要调整配置，可以在 `docker-compose.yml` 中添加环境变量：

```yaml
services:
  api:
    environment:
      # 使用更快的模型（但质量会降低）
      - AI_CUTOUT_REMBG_MODEL=u2netp  # 默认是 u2net（高质量）
      
      # 增加超时时间（如果网络较慢或处理大图）
      - AI_CUTOUT_WORKER_TIMEOUT_MS=120000  # 默认 60000ms
      
      # 配置抠图提供商链
      - AI_CUTOUT_PROVIDER=rembg,mock  # 默认值
```

## ⚠️ 重要提示

### 首次使用
- 首次抠图会自动下载模型文件（约 176MB）
- 下载时间取决于网络速度（通常 30-60 秒）
- 模型下载后会缓存，后续使用无需重新下载

### 网络要求
- 模型下载需要访问 GitHub
- 如果网络受限，可以手动预下载模型（参考文档）

### 内存要求
- 建议至少 2GB 可用内存
- 推荐 3GB 以获得更好的性能
- 如果内存不足，可能导致服务崩溃

### 并发限制
- CPU 版本建议限制并发数为 1-2
- 避免同时处理多个抠图请求导致内存溢出
