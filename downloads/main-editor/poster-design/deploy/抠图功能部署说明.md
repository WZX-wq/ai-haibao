# 抠图功能部署说明

## 📋 概述

当前项目的抠图功能使用 **rembg** 库（基于 U2-Net 深度学习模型）实现本地抠图。

## ⚠️ 重要说明

### 当前部署包状态
**❌ 部署包中不包含抠图功能的完整依赖**

当前的 Docker 镜像（`Dockerfile.api`）**没有**安装：
- Python 运行环境
- rembg 库
- U2-Net 模型文件

### 抠图功能的实现方式

项目中有一个 Python 脚本：`service/scripts/cutout_worker.py`

这个脚本使用 `rembg` 库进行抠图，支持：
- 自动抠图（使用 U2-Net 模型）
- 手动修补擦除
- 蒙版合并

## 🔧 如何启用抠图功能

### 方案 1: 修改 Dockerfile（推荐）

需要修改 `deploy/Dockerfile.api`，添加 Python 和 rembg 支持：

```dockerfile
# 在 FROM node:20-bookworm-slim 之后添加

# 安装 Python 和 pip
RUN apt-get update \
  && apt-get install -y --no-install-recommends \
    python3 \
    python3-pip \
    python3-venv \
    # ... 其他依赖保持不变

# 安装 rembg 和依赖
RUN pip3 install --no-cache-dir \
    rembg[gpu]==2.0.50 \
    pillow \
    onnxruntime

# 或者使用 CPU 版本（更小，但速度较慢）
# RUN pip3 install --no-cache-dir rembg pillow onnxruntime
```

### 方案 2: 使用外部抠图服务

如果不想在本地部署模型，可以：

1. **使用云服务 API**
   - 阿里云视觉智能开放平台
   - 腾讯云图像处理
   - 百度 AI 开放平台
   - Remove.bg API

2. **修改代码调用外部 API**
   - 修改 `src/api/ai.ts` 中的 `cutoutImage` 函数
   - 将请求转发到外部 API 服务

### 方案 3: 单独部署抠图服务

将抠图功能独立部署为微服务：

```dockerfile
# Dockerfile.cutout
FROM python:3.11-slim

WORKDIR /app

RUN pip install --no-cache-dir \
    rembg[gpu]==2.0.50 \
    pillow \
    onnxruntime \
    flask

COPY service/scripts/cutout_worker.py ./
COPY cutout_service.py ./

EXPOSE 8000
CMD ["python", "cutout_service.py"]
```

## 📦 模型文件说明

### rembg 支持的模型

- **u2net** (默认): 176 MB，通用场景
- **u2netp**: 4.7 MB，轻量级版本
- **u2net_human_seg**: 176 MB，人像分割
- **u2net_cloth_seg**: 176 MB，服装分割
- **silueta**: 43 MB，人像轮廓
- **isnet-general-use**: 176 MB，高质量通用
- **isnet-anime**: 176 MB，动漫风格

### 模型下载

**首次运行时自动下载**：
- rembg 会在首次使用时自动从 GitHub 下载模型
- 模型保存在 `~/.u2net/` 目录
- 下载可能需要几分钟（取决于网络速度）

**预下载模型**（可选）：
```bash
# 在容器内或服务器上预先下载
python3 -c "from rembg import new_session; new_session('u2net')"
```

## 📊 性能和资源需求

### CPU 版本
- **内存**: 至少 2GB，推荐 3GB
- **磁盘**: 额外 500MB（模型文件 + 依赖）
- **处理速度**: 5-15秒/张（取决于图片大小）
- **模型大小**: 176MB（u2net 高质量模型）

### GPU 版本（如果有 NVIDIA GPU）
- **内存**: 至少 4GB
- **显存**: 至少 2GB
- **处理速度**: 1-3秒/张
- 需要安装 CUDA 和 cuDNN

## 🔍 故障排查

### 问题 1: 抠图功能不可用

**症状**: 点击抠图按钮没有反应或报错

**解决方案**:
1. 检查 Python 是否已安装：
   ```bash
   docker exec poster_api python3 --version
   ```
   应该显示：Python 3.x.x

2. 检查 rembg 是否已安装：
   ```bash
   docker exec poster_api pip3 list | grep rembg
   ```
   应该显示：rembg 2.0.50

3. 检查 Python 脚本是否存在：
   ```bash
   docker exec poster_api ls -la /app/scripts/cutout_worker.py
   ```
   应该显示文件存在

4. 查看服务日志：
   ```bash
   docker logs poster_api --tail 100
   ```

### 问题 2: 模型下载失败

**症状**: 首次抠图时长时间等待或超时

**可能原因**:
- 网络连接问题（无法访问 GitHub 或模型下载源）
- 防火墙阻止下载
- 容器内存不足

**解决方案**:

**方案 A: 手动预下载模型**
```bash
# 进入容器
docker exec -it poster_api bash

# 手动下载模型
python3 -c "from rembg import new_session; new_session('u2netp')"

# 退出容器
exit
```

**方案 B: 使用国内镜像（如果 GitHub 访问困难）**
```bash
# 在宿主机下载模型文件
wget https://github.com/danielgatis/rembg/releases/download/v0.0.0/u2netp.onnx

# 复制到容器
docker cp u2netp.onnx poster_api:/root/.u2net/u2netp.onnx
```

**方案 C: 增加超时时间**
修改 `docker-compose.yml`，添加环境变量：
```yaml
services:
  api:
    environment:
      - AI_CUTOUT_WORKER_TIMEOUT=120000  # 120秒超时
```

### 问题 3: 内存不足

**症状**: 抠图时服务崩溃或 OOM（Out of Memory）

**解决方案**:

1. 增加容器内存限制（修改 `docker-compose.yml`）：
   ```yaml
   services:
     api:
       deploy:
         resources:
           limits:
             memory: 2G  # 增加到 2GB
   ```

2. 限制并发抠图请求（修改环境变量）：
   ```yaml
   services:
     api:
       environment:
         - AI_CUTOUT_MAX_CONCURRENT=1  # 一次只处理一个抠图请求
   ```

### 问题 4: 抠图速度慢

**症状**: 每次抠图需要 30 秒以上

**原因**: u2netp 是 CPU 版本，速度较慢

**优化方案**:

**方案 A: 使用更快的模型（但质量略低）**
修改环境变量：
```yaml
services:
  api:
    environment:
      - AI_CUTOUT_REMBG_MODEL=u2netp  # 已经是最快的模型
```

**方案 B: 升级到 GPU 版本（需要 NVIDIA GPU）**
修改 Dockerfile.api：
```dockerfile
# 将 onnxruntime 替换为 GPU 版本
RUN pip3 install --no-cache-dir \
    rembg==2.0.50 \
    pillow==10.1.0 \
    onnxruntime-gpu==1.16.3
```

然后在 docker-compose.yml 中启用 GPU：
```yaml
services:
  api:
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: 1
              capabilities: [gpu]
```

**方案 C: 使用云服务 API（推荐用于高并发场景）**
参考下方"方案 2: 使用外部抠图服务"

### 问题 5: 抠图质量不佳

**症状**: 边缘不清晰、主体不完整

**原因**: u2netp 是轻量级模型，质量中等

**解决方案**:

切换到更高质量的模型（但速度会变慢）：
```yaml
services:
  api:
    environment:
      - AI_CUTOUT_REMBG_MODEL=u2net  # 使用完整版模型（176MB）
```

或者：
```yaml
services:
  api:
    environment:
      - AI_CUTOUT_REMBG_MODEL=isnet-general-use  # 最高质量（176MB）
```

**注意**: 更换模型后需要重新下载模型文件。

## 💡 优化建议

### 1. 使用模型缓存
```python
# 在服务启动时预加载模型
session = new_session('u2net')
```

### 2. 添加请求队列
限制同时处理的抠图请求数量，避免内存溢出。

### 3. 使用 CDN
将抠图结果上传到 CDN，避免重复处理。

### 4. 异步处理
使用消息队列（如 Redis）实现异步抠图，提升用户体验。
