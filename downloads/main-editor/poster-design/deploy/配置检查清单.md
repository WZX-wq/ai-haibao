## 📋 部署前检查

### 必须配置的环境变量
在 `deploy/.env` 文件中，以下变量必须配置：

```bash
# 数据库配置（必填）
DB_HOST=your_mysql_host
DB_PORT=3306
DB_NAME=poster_design
DB_USERNAME=your_username
DB_PASSWORD=your_password

# OAuth 配置（如果启用统一登录）
OAUTH_CLIENT_ID=your_client_id
OAUTH_CLIENT_SECRET=your_client_secret
OAUTH_REDIRECT_URI=https://your-domain.com/oauth/callback

# 百炼 API 密钥（如果使用 AI 功能）
AI_BAILIAN_API_KEY=your_api_key
```

### 可选配置的环境变量
```bash
# 抠图功能配置（已有默认值）
AI_CUTOUT_REMBG_MODEL=u2net  # 默认高质量模型
AI_CUTOUT_PYTHON=python3  # 默认 Python 3
AI_CUTOUT_WORKER_TIMEOUT_MS=60000  # 默认 60 秒

# 如果需要更快速度但质量略低
# AI_CUTOUT_REMBG_MODEL=u2netp

# 如果需要最高质量但速度较慢
# AI_CUTOUT_REMBG_MODEL=isnet-general-use
```

## 🚀 部署步骤

### 1. 解压部署包
```bash
unzip poster-design-docker-deploy.zip
cd poster-design/deploy
```

### 2. 配置环境变量
```bash
cp env.template .env
# 编辑 .env 文件，填写必要的配置
```

### 3. 初始化数据库
在 MySQL 中执行 `mysql/init` 目录下的 SQL 文件。

### 4. 构建并启动服务
```bash
docker compose -f docker-compose.yaml up -d --build
```

### 5. 验证部署
- 访问 `http://your-domain.com`
- 测试 PC 端功能
- 测试移动端响应式布局
- 测试抠图功能（首次会下载模型）

### 抠图功能资源需求
- **额外内存**: 2-3GB（u2net 模型运行时）

- **首次下载**: 176MB（u2net 模型）

## ⚠️ 重要提示

### 1. 首次抠图
- 首次使用抠图功能会自动下载 u2net 模型（176MB）
- 下载时间：30-60 秒（取决于网络速度）
- 下载完成后会缓存，后续使用无需重新下载

### 2. 内存管理
- 建议为 Docker 容器分配至少 3GB 内存
- 如果内存不足，可以切换到 u2netp 模型（5MB，但质量略低）

### 3. 网络要求
- 模型下载需要访问 GitHub
- 如果网络受限，可以手动预下载模型（参考文档）

### 4. PC 端兼容性
- 所有移动端修改都使用 `@media (max-width: 1023px)` 隔离
- PC 端样式完全不受影响
- 已经过充分测试，确保 PC 端功能正常

## 🔍 故障排查

### 问题 1: 抠图功能不可用
```bash
# 检查 Python 是否安装
docker exec poster_api python3 --version

# 检查 rembg 是否安装
docker exec poster_api pip3 list | grep rembg

# 检查脚本是否存在
docker exec poster_api ls -la /app/scripts/cutout_worker.py

# 查看日志
docker logs poster_api --tail 100
```

### 问题 2: 模型下载失败
```bash
# 手动预下载模型
docker exec -it poster_api bash
python3 -c "from rembg import new_session; new_session('u2net')"
exit
```

### 问题 3: 内存不足
修改 `docker-compose.yaml`：
```yaml
services:
  api:
    deploy:
      resources:
        limits:
          memory: 3G
```

### 问题 4: 移动端布局异常
- 清除浏览器缓存
- 检查是否使用了正确的部署包
- 验证 responsive.less 文件是否存在
